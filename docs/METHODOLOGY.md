# Methodology

## Overview

Each TAI KIT item originates as a **single-view photograph** of a generic
kitchen object. The photograph is processed through a multi-stage neural
pipeline to produce a textured 3D mesh.

```
photo → preprocess → bg removal → image-to-3D → postprocess → glTF
```

## Pipeline stages

### 1. Preprocessing
Source images are upscaled (LANCZOS 4×) from low-resolution thumbnails to
provide a larger input canvas for downstream feature extraction.

### 2. Background removal
The object is isolated from its background using
[BiRefNet](https://github.com/ZhengPeng7/BiRefNet) (MIT-licensed variant).
This produces a clean subject mask and prevents the reconstructor from
hallucinating ground or environment geometry.

### 3. Image-to-3D reconstruction
The masked image is passed to
[TRELLIS.2](https://github.com/microsoft/TRELLIS.2) (Microsoft, MIT) which
produces a textured mesh via sparse-voxel latent diffusion. TRELLIS.2
internally uses [DINOv3](https://github.com/facebookresearch/dinov3)
features for image conditioning.

### 4. Postprocessing
- A slab-cut check detects residual floor-plate artifacts (when the bottom
  5% of the Y-axis contains more than 30% of the total face count) and
  removes them.
- The mesh is re-exported through `trimesh` to strip tool-specific metadata
  before release.

## Known limitations

- **Back-side hallucination**: single-view input cannot observe the rear of
  the object. Reconstructed back surfaces are plausible but speculative.
- **Texture fragmentation**: UV islands generated by the sparse-voxel
  pipeline can be fragmented, occasionally producing seams or salt-pepper
  artifacts under simple per-face shading. They are best viewed with the
  included PBR textures.
- **Scale is approximate**: dimensions in `metadata.json` are best-effort
  from source metadata. For exact metric scale, calibration is advised.

## Hardware & runtime

Reconstruction of the parent dataset (from which this 100-item release is
sampled) was performed on 8× NVIDIA H100 SXM GPUs for approximately 17
compute hours.

## Why open source

Open 3D datasets of everyday objects — especially non-Western kitchenware —
are scarce. We release TAI KIT as a small-scale test release to seed
research, education, and creative work, and to invite feedback before
expanding to a fuller collection.
